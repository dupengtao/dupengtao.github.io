怎样保持fork出的项目和上游项目保持更新?

       在日常工作中，越来越多要使用开源组件，如联网、图片处理、自定义控件等。clone一份代码到本地，根据自己的需求修改并使用后就不去管它了，这样做是非常不好的，应该经常去同步这个库，使它变得更完善。
       fork出一个项目后，如果原有的项目更新了，怎样使我们fork出来的项目和原有项目保持同步？怎样提交我们的代码更新？即怎样保持fork出的项目和上游项目保持更新。

以github举栗：

1.先fork一下开源组件，并clone到本地

2.保持本地master分支与远程上游master分支一致，命令只需执行一次即可。upstream 可任意起名称

这里执行了rebase操作，上游最新的节点更新到本地master分支上了。到这里已经完成了与上游项目更新了。

本地开发相关：
3.创建一个开发分支，平常本地修改都提交这

4.本地采用merge/rebase/cherry-pick 等方式把所需要的内容合并到本地开发分支，并提交到远程分支（需要本地开发分支与远程开发分支先进行关联）
